# Eval 命令

管理基于评估的开发workflow process。

## 使用方法

`/eval [define|check|report|list] [功能名称]`

## 定义评估

`/eval define 功能名称`

创建新的评估定义：

1. 创建 `.claude/evals/功能名称.md`，包含模板：

```markdown
## 评估：功能名称
创建于：$(date)

### 能力评估
- [ ] [能力 1 的描述]
- [ ] [能力 2 的描述]

### 回归评估
- [ ] [现有行为 1 仍然工作]
- [ ] [现有行为 2 仍然工作]

### 成功标准
- pass@3 > 90% 用于能力评估
- pass^3 = 100% 用于回归评估
```

2. 提示用户填写具体标准

## 检查评估

`/eval check 功能名称`

运行功能的评估：

1. 从 `.claude/evals/功能名称.md` 读取评估定义
2. 对于每个能力评估：
   - 尝试验证标准
   - 记录通过/失败
   - 在 `.claude/evals/功能名称.log` 中记录尝试
3. 对于每个回归评估：
   - 运行相关测试
   - 与基线比较
   - 记录通过/失败
4. 报告当前状态：

```
评估检查：功能名称
========================
能力：X/Y 通过
回归：X/Y 通过
状态：进行中 / 就绪
```

## 评估报告

`/eval report 功能名称`

生成综合评估报告：

```
评估报告：功能名称
=========================
生成于：$(date)

能力评估
----------------
[评估-1]：通过（pass@1）
[评估-2]：通过（pass@2）- 需要重试
[评估-3]：失败 - 见备注

回归评估
----------------
[测试-1]：通过
[测试-2]：通过
[测试-3]：通过

指标
-------
能力 pass@1：67%
能力 pass@3：100%
回归 pass^3：100%

备注
-----
[任何问题、边缘情况或观察]

建议
--------------
[发布 / 需要工作 / 阻塞]
```

## 列出评估

`/eval list`

显示所有评估定义：

```
评估定义
================
feature-auth      [3/5 通过] 进行中
feature-search    [5/5 通过] 就绪
feature-export    [0/4 通过] 未启动
```

## 参数

$ARGUMENTS：
- `define <名称>` - 创建新的评估定义
- `check <名称>` - 运行并检查评估
- `report <名称>` - 生成完整报告
- `list` - 显示所有评估
- `clean` - 清除旧的评估日志（保留最后 10 次运行）
