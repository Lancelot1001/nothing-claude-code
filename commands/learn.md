# /learn - 提取可重用模式

分析当前会话并提取任何值得保存为skill的模式。

## 触发

在任何时候当你解决了一个非平凡问题时，运行 `/learn`。

## 提取什么

寻找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这是否可用于类似错误？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方案**
   - 库的问题
   - API 限制
   - 版本特定修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建skill文件：

```markdown
# [描述性模式名称]

**提取日期：** [日期]
**上下文：** [适用场景的简要描述]

## 问题
[解决的问题 — 要具体]

## 解决方案
[模式/技术/变通方案]

## 示例
[如有代码示例]

## 何时使用
[触发条件 — 什么应该激活此skill]
```

## 流程

1. 回顾会话中可提取的模式
2. 识别最有价值/可重用的见解
3. 起草skill文件
4. 保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不提取平凡修复（拼写错误、简单语法错误）
- 不提取一次性问题（特定 API 中断等）
- 专注于未来会话中节省时间的模式
- 保持skill专注 — 每个skill一个模式
