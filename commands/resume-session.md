# 恢复会话命令

加载最后保存的会话状态，并在做任何工作之前完全定向。
此命令是 `/save-session` 的对应命令。

## 何时使用

- 开始新会话以继续前一天的工作
- 由于上下文限制而开始新会话后
- 从其他来源交接会话文件时（只需提供文件路径）
- 任何时候你有会话文件并希望 Claude 在继续之前完全吸收它

## 使用方式

```
/resume-session                                                      # 加载 ~/.claude/session-data/ 中最近的文件
/resume-session 2024-01-15                                           # 加载该日期最近的会话
/resume-session ~/.claude/session-data/2024-01-15-abc123de-session.tmp  # 加载带短 ID 的当前会话文件
/resume-session ~/.claude/sessions/2024-01-15-session.tmp               # 加载特定旧格式文件
```

## 流程

### 步骤 1：找到会话文件

如未提供参数：

1. 检查 `~/.claude/session-data/`
2. 选择最近修改的 `*-session.tmp` 文件
3. 如文件夹不存在或无匹配文件，告知用户：
   ```
   在 ~/.claude/session-data/ 中未找到会话文件
   运行 /save-session 在会话结束时创建一个。
   ```
   然后停止。

如提供了参数：

- 如看起来像日期（`YYYY-MM-DD`），先搜索 `~/.claude/session-data/`，然后搜索旧版
  `~/.claude/sessions/`，查找匹配 `YYYY-MM-DD-session.tmp`（旧格式）或
  `YYYY-MM-DD-<shortid>-session.tmp`（当前格式）的文件，并加载该日期最近修改的变体
- 如看起来像文件路径，直接读取该文件
- 如未找到，清楚报告并停止

### 步骤 2：读取整个会话文件

读取完整文件。尚不总结。

### 步骤 3：确认理解

用此确切格式回复结构化简报：

```
SESSION LOADED: [实际解析的文件路径]
════════════════════════════════════════════════

PROJECT: [项目名称/主题]

我们在构建什么：
[2-3 句你自己的总结]

当前状态：
PASS: 工作中：[已确认] 项
 进行中：[正在进行中的文件列表]
 尚未开始：[已计划但未触摸的]

不应重试的：
[列出每个失败方法及其原因 — 这很关键]

开放问题/障碍：
[列出任何障碍或未回答的问题]

下一个步骤：
[如文件中定义：确切的下一个步骤]
[如未定义："下一个步骤未定义 — 建议在开始前一起审查'尚未尝试的内容'"]

════════════════════════════════════════════════
准备好继续了。你想做什么？
```

### 步骤 4：等待用户

不自动开始工作。不触摸任何文件。等待用户说明下一步做什么。

如会话文件中明确定义了下一个步骤，且用户说"继续"或"是"或类似 — 用该确切下一步继续。

如未定义下一步 — 询问用户从哪里开始，并可选择从"尚未尝试什么"章节建议方法。

## 边缘情况

**同一日期的多个会话**（`2024-01-15-session.tmp`、`2024-01-15-abc123de-session.tmp`）：
加载该日期最近修改的匹配文件，无论它使用旧无 ID 格式还是当前短 ID 格式。

**会话文件引用不再存在的文件：**
在简报中注明 — "警告：`path/to/file.ts` 在会话中引用但磁盘上未找到。"

**会话文件来自 7 天多前：**
注明差距 — "警告：此会话来自 N 天前（阈值：7 天）。情况可能已变更。" — 然后正常继续。

**用户直接提供文件路径（例如从队友转发）：**
读取并遵循相同的简报流程 — 无论来源如何，格式相同。

**会话文件为空或格式错误：**
报告："找到会话文件但似乎为空或不可读。你可能需要用 /save-session 创建一个新的。"

## 注意事项

- 加载时绝不修改会话文件 — 它是只读的历史记录
- 简报格式是固定的 — 即使章节为空也不要跳过
- "不应重试"必须始终显示，即使仅写"无" — 这太重要了不能遗漏
- 恢复后，用户可能希望在新的会话结束时再次运行 `/save-session` 以创建新的带日期文件
