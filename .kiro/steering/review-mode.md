# 审查模式

进行代码审查或质量评估时使用此上下文。

## 审查流程

1. 收集上下文——检查 git diff 查看所有变更
2. 理解范围——识别哪些文件改变了以及为什么
3. 阅读周围代码——不孤立审查
4. 应用审查清单——逐项检查
5. 报告发现——使用严重级别

## 审查清单

### 正确性
- 代码做了它应该做的吗？
- 边界情况处理得当吗？
- 错误处理适当吗？

### 安全
- 输入验证和净化了吗？
- secret 管理得当吗？
- 有注入漏洞吗？
- 认证/授权正确吗？

### 性能
- 有明显的性能问题吗？
- 数据库查询优化了吗？
- 缓存使用适当吗？

### 可维护性
- 代码可读且组织良好吗？
- 函数和类大小适当吗？
- 有足够的文档吗？
- 命名约定被遵循了吗？

### 测试
- 有足够的测试吗？
- 测试覆盖了边界情况吗？
- 测试清晰可维护吗？

## 严重级别

- **Critical**：安全漏洞、数据丢失风险
- **High**：破坏功能的 bug、重大性能问题
- **Medium**：代码质量问题、可维护性担忧
- **Low**：风格不一致、小的改进

## 调用

审查代码时使用 `#review-mode` 激活此上下文。
