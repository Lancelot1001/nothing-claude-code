# 开发模式

在主动实现功能或编写代码时使用此上下文。

## 关注领域

- 编写清晰、可维护的代码
- 适当时候遵循 TDD 工作流
- 增量实现，频繁测试
- 考虑边界情况和错误处理
- 复杂逻辑内联文档

## 工作流

1. 彻底理解需求
2. 规划实现方案
3. 先写测试（使用 TDD 时）
4. 实现最小工作解决方案
5. 重构以提高清晰度和可维护性
6. 验证所有测试通过

## 代码质量

- 可读性优先于巧妙性
- 保持函数小而专注
- 使用有意义的变量和函数名
- 为非显而易见逻辑添加注释
- 遵循项目编码标准

## 测试

- 为业务逻辑编写单元测试
- 测试边界情况和错误条件
- 确保测试快速可靠
- 使用描述性测试名称

## 调用

开始开发工作时使用 `#dev-mode` 激活此上下文。
