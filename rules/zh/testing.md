# 测试要求

## 最低覆盖率：80%

**三种测试类型（全部需要）**：
1. **单元测试** — 独立函数、工具、组件
2. **集成测试** — API 端点、数据库操作
3. **E2E 测试** — 关键用户流程

## 测试驱动开发（TDD）

**强制工作流**：
1. 先写测试（RED）— 测试应该**失败**
2. 运行测试 — 验证失败原因是正确的
3. 写最小实现（GREEN）— 测试应该**通过**
4. 运行测试 — 验证通过
5. 重构（IMPROVE）— 在保持测试绿色时改进
6. 验证覆盖率 — 确保 80%+

## 故障排除

测试失败时：
1. 使用 **tdd-guide** agent
2. 检查测试隔离
3. 验证 mock 正确
4. 修复实现，而非测试（除非测试本身有误）

## Agent 支持

- **tdd-guide** — 新功能时主动使用，强制先写测试
