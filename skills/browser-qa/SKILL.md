# 浏览器 QA — 自动化可视化测试与交互

## 何时使用

- 将功能部署到 staging/preview 后
- 当你需要跨页面验证 UI 行为时
- 发货前 — 确认布局、表单、交互实际可用
- 审查涉及前端代码的 PR 时
- 无障碍审计和响应式测试

## 工作原理

使用浏览器自动化 MCP（claude-in-chrome、Playwright 或 Puppeteer）与真实页面交互，如同真实用户。

### 阶段 1：冒烟测试
```
1. 导航到目标 URL
2. 检查控制台错误（过滤噪音：分析工具、第三方）
3. 验证网络请求中没有 4xx/5xx
4. 在桌面和移动视口上截图首屏内容
5. 检查 Core Web Vitals：LCP < 2.5s，CLS < 0.1，INP < 200ms
```

### 阶段 2：交互测试
```
1. 点击每个导航链接 — 验证没有死链接
2. 用有效数据提交表单 — 验证成功状态
3. 用无效数据提交表单 — 验证错误状态
4. 测试认证流程：登录 → 受保护页面 → 登出
5. 测试关键用户旅程（结账、入职、搜索）
```

### 阶段 3：视觉回归
```
1. 在 3 个断点（375px、768px、1440px）截图关键页面
2. 与基准截图比较（如有存储）
3. 标记布局偏移 > 5px、缺失元素、溢出
4. 检查暗色模式（如适用）
```

### 阶段 4：无障碍
```
1. 在每个页面运行 axe-core 或等效工具
2. 标记 WCAG AA 违规（对比度、标签、焦点顺序）
3. 验证键盘导航端到端可用
4. 检查屏幕阅读器地标
```

## 输出格式

```markdown
## QA 报告 — [URL] — [时间戳]

### 冒烟测试
- 控制台错误：0 个严重，2 个警告（分析工具噪音）
- 网络：全部 200/304，无失败
- Core Web Vitals：LCP 1.2s ✓，CLS 0.02 ✓，INP 89ms ✓

### 交互
- [✓] 导航链接：12/12 可用
- [✗] 联系表单：无效邮箱时缺少错误状态
- [✓] 认证流程：登录/登出可用

### 视觉
- [✗] 首屏区域在 375px 视口上溢出
- [✓] 暗色模式：所有页面一致

### 无障碍
- 2 个 AA 违规：首屏图片缺少 alt 文本，页脚链接对比度低

### 判断：修复后发货（2 个问题，0 个阻塞项）
```

## 集成

可与任何浏览器 MCP 配合使用：
- `mChild__claude-in-chrome__*` 工具（首选 — 使用你实际的 Chrome）
- 通过 `mcp__browserbase__*` 使用 Playwright
- 直接 Puppeteer 脚本

与 `/canary-watch` 配合用于部署后监控。
