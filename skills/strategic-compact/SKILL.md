# 战略压缩skill

建议在您workflow的战略时点进行手动的 `/compact`，而非依赖任意的自动压缩。

## 何时激活

- 运行接近上下文限制的长会话（200K+ tokens）
- 处理多阶段任务（研究 → 计划 → 实现 → 测试）
- 在同一会话中切换不相关的任务
- 完成一个重要里程碑并开始新工作后
- 当响应变慢或变得不那么连贯时（上下文压力）

## 为什么需要战略压缩？

自动压缩在任意时点触发：
- 经常在任务中途发生，丢失重要上下文
- 无法感知逻辑任务边界
- 可能中断复杂的多步骤操作

在逻辑边界进行战略压缩：
- **探索后、执行前** — 压缩研究上下文，保留实现计划
- **完成里程碑后** — 为下一阶段全新开始
- **主要上下文切换前** — 在不同任务之前清除探索上下文

## 工作原理

`suggest-compact.js` 脚本在 PreToolUse（Edit/Write）上运行：

1. **追踪工具调用** — 计算会话中的工具调用次数
2. **阈值检测** — 在可配置阈值（默认：50 次调用）时建议
3. **定期提醒** — 达到阈值后每 25 次调用提醒一次

## 钩子设置

添加到您的 `~/.claude/settings.json`：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      },
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      }
    ]
  }
}
```

## 配置

环境变量：
- `COMPACT_THRESHOLD` — 首次建议前的工具调用次数（默认：50）

## 压缩决策指南

使用此表决定何时压缩：

| 阶段转换 | 压缩？ | 原因 |
|-----------------|----------|-----|
| 研究 → 计划 | 是 | 研究上下文冗长；计划是提炼后的输出 |
| 计划 → 实现 | 是 | 计划已在 TodoWrite 或文件中；释放上下文用于代码 |
| 实现 → 测试 | 可能 | 如果测试引用了最近代码则保留；如果切换焦点则压缩 |
| 调试 → 下一个功能 | 是 | 调试追踪会污染不相关工作的上下文 |
| 实现中途 | 否 | 丢失变量名、文件路径和部分状态代价高昂 |
| 失败的方法之后 | 是 | 在尝试新方法前清除死胡同推理 |

## 什么在压缩后保留

了解什么会保留帮助您有信心地进行压缩：

| 保留 | 丢失 |
|----------|------|
| CLAUDE.md 指令 | 中间推理和分析 |
| TodoWrite 任务列表 | 您之前读取的文件内容 |
| 内存文件（`~/.claude/memory/`） | 多步骤对话上下文 |
| Git 状态（提交、分支） | 工具调用历史和计数 |
| 磁盘上的文件 | 口头表达的用户偏好细节 |

## 最佳实践

1. **计划后压缩** — 一旦计划在 TodoWrite 中定稿，压缩以全新开始
2. **调试后压缩** — 在继续前清除错误解决上下文
3. **不要在实现中途压缩** — 为相关更改保留上下文
4. **阅读建议** — 钩子告诉您*何时*，您决定*是否*
5. **压缩前写入** — 在压缩前将重要上下文保存到文件或内存
6. **使用带摘要的 `/compact`** — 添加自定义消息：`/compact 专注于实现 auth 中间件`

## Token 优化模式

### 触发表延迟加载
不是在会话开始时加载完整skill内容，而是使用将关键字映射到skill路径的触发表。skill仅在触发时加载，减少基线上下文 50%+：

| 触发词 | skill | 加载时机 |
|---------|-------|-----------|
| "test", "tdd", "coverage" | tdd-workflow | 用户提到测试时 |
| "security", "auth", "xss" | security-review | 安全相关工作时 |
| "deploy", "ci/cd" | deployment-patterns | 部署上下文 |

### 上下文组成意识
监控消耗您上下文窗口的内容：
- **CLAUDE.md 文件** — 始终加载，保持精简
- **已加载skill** — 每个skill增加 1-5K tokens
- **对话历史** — 每次交互增长
- **工具结果** — 文件读取、搜索结果增加体积

### 重复指令检测
重复上下文的常见来源：
- `~/.claude/rules/` 和项目 `.claude/rules/` 中的相同规则
- skill重复 CLAUDE.md 指令
- 多个skill涵盖重叠领域

### 上下文优化工具
- `token-optimizer` MCP — 通过内容去重自动减少 95%+ tokens
- `context-mode` — 上下文虚拟化（已演示 315KB 到 5.4KB）
