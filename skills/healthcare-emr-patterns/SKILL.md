# 医疗 EMR 开发模式

用于构建电子病历（EMR）和电子健康记录（EHR）系统的模式。优先考虑患者安全、临床准确性和从业者效率。

## 何时使用

- 构建患者就诊workflow（主诉、检查、诊断、处方）
- 实现临床笔记（结构化 + 免费文本 + 语音转文本）
- 设计带药物相互作用检查的处方/药物模块
- 集成临床决策支持系统（CDSS）
- 构建带参考范围高亮的实验室结果显示
- 为临床数据实现审计跟踪
- 为医疗数据录入设计无障碍 UI

## 工作原理

### 患者安全第一

每个设计决策都必须评估："这会伤害患者吗？"

- 药物相互作用必须警报，而非静默通过
- 异常实验室值必须视觉标记
- 危重生命体征必须触发升级workflow
- 所有临床数据修改必须有审计跟踪

### 单页就诊流程

临床就诊应在单页上垂直流动——无标签切换：

```
患者头部（粘性——始终可见）
├── 人口统计学、过敏、活跃用药
│
就诊流程（垂直滚动）
├── 1. 主诉（结构化模板 + 免费文本）
├── 2. 现病史
├── 3. 体格检查（按系统）
├── 4. 生命体征（自动触发临床评分）
├── 5. 诊断（ICD-10/SNOMED 搜索）
├── 6. 用药（药物数据库 + 相互作用检查）
├── 7. 检查（实验室/影像医嘱）
├── 8. 计划与随访
└── 9. 签署/锁定/打印
```

### 智能模板系统

```typescript
interface ClinicalTemplate {
  id: string;
  name: string;             // 如 "Chest Pain"
  chips: string[];          // 可点击症状芯片
  requiredFields: string[]; // 强制数据点
  redFlags: string[];       // 触发不可关闭警报
  icdSuggestions: string[]; // 预映射诊断代码
}
```

任何模板中的红旗必须触发可见的、不可关闭的警报——而非 toast 通知。

### 药物安全模式

```
用户选择药物
  → 检查当前用药的相互作用
  → 检查就诊用药的相互作用
  → 检查患者过敏
  → 根据体重/年龄/肾功能验证剂量
  → 如果是危重相互作用：完全阻止处方
  → 临床医生必须记录覆盖原因才能继续
  → 如果是主要相互作用：显示警告，要求确认
  → 将所有警报和覆盖原因记录到审计跟踪
```

危重相互作用**默认阻止处方**。临床医生必须用记录的原因明确覆盖，系统永远不会静默允许危重相互作用。

### 锁定就诊模式

一旦临床就诊被签署：
- 不允许编辑——仅允许附录（单独的关联记录）
- 原始记录和附录都显示在患者时间线上
- 审计跟踪捕获签署人、时间和任何附录记录

### 临床数据的 UI 模式

**生命体征显示：** 当前值带正常范围高亮（绿/黄/红）、与前值对比的趋势箭头、自动计算的临床评分（NEWS2、qSOFA）、内联升级指导。

**实验室结果显示：** 正常范围高亮、前值对比、危重值带不可关闭警报、采集/分析时间戳、待定医嘱及预期周转时间。

**处方 PDF：** 一键生成，包含患者人口统计学、过敏、诊断、药物详情（通用名 + 品牌、剂量、途径、频率、疗程）、临床医生签名栏。

### 医疗无障碍

医疗 UI 比典型 Web 应用有更严格要求：
- 4.5:1 最低对比度（WCAG AA）——临床医生在各种光照条件下工作
- 大触摸目标（44x44px 最低）——用于戴手套/匆忙的交互
- 键盘导航——用于快速录入数据的power用户
- 非纯色指示器——始终配合颜色使用文本/图标（色盲临床医生）
- 所有表单字段的屏幕阅读器标签
- 临床警报无自动关闭 toast——临床医生必须主动确认

### 反模式

- 在浏览器 localStorage 中存储临床数据
- 药物相互作用检查中的静默失败
- 临床警报的可关闭 toast
- 基于标签的就诊 UI 破坏临床workflow
- 允许编辑已签署/锁定的就诊
- 显示临床数据无审计跟踪
- 对临床数据结构使用 `any` 类型

## 示例

### 示例 1：患者就诊流程

```
医生为患者 #4521 打开就诊
  → 粘性头部显示："Rajesh M, 58M, 过敏：青霉素，活跃用药：二甲双胍 500mg"
  → 主诉：选择 "Chest Pain" 模板
    → 点击芯片："substernal"、"radiating to left arm"、"crushing"
    → 红旗 "crushing substernal chest pain" 触发不可关闭警报
  → 检查：心血管系统——"S1 S2 正常，无杂音"
  → 生命体征：HR 110, BP 90/60, SpO2 94%
    → NEWS2 自动计算：评分 8，高风险，升级警报显示
  → 诊断：搜索 "ACS" → 选择 ICD-10 I21.9
  → 用药：选择阿司匹林 300mg
    → CDSS 检查二甲双胍：无相互作用
  → 签署就诊 → 锁定，此后仅允许附录
```

### 示例 2：药物安全workflow

```
医生为患者 #4521 开具华法林
  → CDSS 检测到：华法林 + 阿司匹林 = 危重相互作用
  → UI：红色不可关闭模态框阻止处方
  → 医生点击"附原因覆盖"
  → 输入："获益大于风险——监测 INR 方案"
  → 覆盖原因 + 警报存储在审计跟踪中
  → 处方继续，记录的覆盖
```

### 示例 3：锁定就诊 + 附录

```
就诊 #E-2024-0891 由 Dr. Shah 于 14:30 签署
  → 所有字段锁定——无编辑按钮可见
  → "添加附录"按钮可用
  → Dr. Shah 点击附录，添加："收到实验室结果——肌钙蛋白升高"
  → 新记录 E-2024-0891-A1 关联到原始记录
  → 时间线显示两者：原始就诊 + 带时间戳的附录
```
