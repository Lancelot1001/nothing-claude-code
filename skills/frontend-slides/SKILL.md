# 前端幻灯片

创建零依赖、动画丰富的 HTML 演示文稿，完全在浏览器中运行。

灵感来自 zarazhangrui 作品中展示的视觉探索方法（鸣谢：@zarazhangrui）。

## 何时激活

- 创建演讲稿、推介稿、工作坊稿或内部演示
- 将 `.ppt` 或 `.ppttx` 幻灯片转换为 HTML 演示文稿
- 改进现有 HTML 演示文稿的布局、动效或排版
- 与尚不了解设计偏好的用户探索演示风格

## 不可妥协的要求

1. **零依赖**：默认使用一个自包含的 HTML 文件，内联 CSS 和 JS。
2. **视口适配是强制性的**：每张幻灯片必须适配一个视口，无内部滚动。
3. **展示而非说教**：使用视觉预览而非抽象的风格问卷。
4. **独特的设计**：避免通用的紫色渐变、白色 Inter 字体、模板化的演示文稿。
5. **生产质量**：保持代码有注释、无障碍、响应式且高性能。

在生成前，阅读 `STYLE_PRESETS.md` 了解视口安全的 CSS 基础、密度限制、预设目录和 CSS 陷阱。

## workflow

### 1. 检测模式

选择一条路径：
- **新演示文稿**：用户有主题、笔记或完整草稿
- **PPT 转换**：用户有 `.ppt` 或 `.pptx`
- **增强**：用户已有 HTML 幻灯片并希望改进

### 2. 发现内容

仅询问最少必要信息：
- 目的：推介、教学、会议演讲、内部更新
- 长度：短（5-10）、中（10-20）、长（20+）
- 内容状态：完成稿、粗糙笔记、仅主题

如果用户有内容，请在样式化前让他们粘贴。

### 3. 发现风格

默认进行视觉探索。

如果用户已经知道想要的预设，跳过预览直接使用。

否则：
1. 询问演示文稿应该创造什么感觉：令人印象深刻、充满活力、专注、受启发。
2. 在 `.ecc-design/slide-previews/` 中生成 **3 个单幻灯片预览文件**。
3. 每个预览必须自包含，清楚展示字体/颜色/动效，且保持在约 100 行幻灯片内容以内。
4. 询问用户保留哪个预览或混合哪些元素。

使用 `STYLE_PRESETS.md` 中的预设指南将情绪映射到风格。

### 4. 构建演示文稿

输出以下之一：
- `presentation.html`
- `[presentation-name].html`

仅当演示文稿包含提取的或用户提供的图像时才使用 `assets/` 文件夹。

必需结构：
- 语义化幻灯片 section
- 来自 `STYLE_PRESETS.md` 的视口安全 CSS 基础
- 主题值的 CSS 自定义属性
- 用于键盘、滚轮和触摸导航的演示控制器类
- 用于揭示动画的 Intersection Observer
- 减少动效支持

### 5. 强制视口适配

将其视为硬性门槛。

规则：
- 每个 `.slide` 必须使用 `height: 100vh; height: 100dvh; overflow: hidden;`
- 所有类型和间距必须使用 `clamp()` 缩放
- 当内容不适配时，分割为多张幻灯片
- 绝不通过将文本缩小到可读大小以下来解决溢出问题
- 绝不允许幻灯片内出现滚动条

使用 `STYLE_PRESETS.md` 中的密度限制和强制 CSS 块。

### 6. 验证

在这些尺寸下检查完成的演示文稿：
- 1920x1080
- 1280x720
- 768x1024
- 375x667
- 667x375

如果浏览器自动化可用，使用它验证无幻灯片溢出且键盘导航正常工作。

### 7. 交付

在交接时：
- 删除临时预览文件，除非用户想保留
- 在有用时使用平台适当的方式打开演示文稿
- 总结文件路径、使用的预设、幻灯片数量和简单的主题定制点

使用当前 OS 的正确打开方式：
- macOS：`open file.html`
- Linux：`xdg-open file.html`
- Windows：`start "" file.html`

## PPT/PPTX 转换

关于 PowerPoint 转换：
1. 优先使用 `python3` 和 `python-pptx` 提取文本、图像和备注。
2. 如果 `python-pptx` 不可用，询问是否安装或回退到手动/导出workflow。
3. 保留幻灯片顺序、演讲者备注和提取的资源。
4. 提取后，运行与新演示文稿相同的选择风格workflow。

保持转换跨平台。当 Python 可以完成工作时，不要依赖 macOS 专用工具。

## 实现要求

### HTML/CSS

- 除非用户明确想要多文件项目，否则使用内联 CSS 和 JS。
- 字体可以来自 Google Fonts 或 Fontshare。
- 优先使用有氛围感的背景、强烈的字体层次和清晰的视觉方向。
- 使用抽象形状、渐变、网格、噪点和几何图形，而非插图。

### JavaScript

包含：
- 键盘导航
- 触摸/滑动导航
- 鼠标滚轮导航
- 进度指示器或幻灯片索引
- 进入时触发揭示动画

### 无障碍

- 使用语义结构（`main`、`section`、`nav`）
- 保持对比度可读
- 支持纯键盘导航
- 尊重 `prefers-reduced-motion`

## 内容密度限制

除非用户明确要求更密集的幻灯片且仍保持可读性，否则使用以下最大值：

| 幻灯片类型 | 限制 |
|------------|------|
| 标题 | 1 个标题 + 1 个副标题 + 可选标语 |
| 内容 | 1 个标题 + 4-6 个要点或 2 个短段落 |
| 特性网格 | 最多 6 张卡片 |
| 代码 | 最多 8-10 行 |
| 引言 | 1 条引言 + 作者归属 |
| 图像 | 1 张受视口约束的图像 |

## 反模式

- 无视觉标识的通用创业公司渐变
- 除非刻意编辑，否则使用系统字体
- 长篇要点墙
- 需要滚动的代码块
- 在短屏幕上破坏的固定高度内容框
- 无效的否定 CSS 函数如 `-clamp(...)`
