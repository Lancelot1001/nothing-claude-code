# 内容引擎

将一个想法转化为强大的平台原生内容，而不是在各处发布相同的内容。

## 何时激活

- 撰写 X 帖子或主题帖
- 起草 LinkedIn 帖子或发布更新
- 编写短视频或 YouTube 解释视频脚本
- 将文章、播客、演示或文档重新用于社交内容
- 围绕发布、里程碑或主题构建轻量级内容计划

## 初步问题

先明确：
- 源素材：我们从什么改编
- 受众：建设者、投资者、客户、运营商还是普通受众
- 平台：X、LinkedIn、TikTok、YouTube、新闻通讯还是多平台
- 目标：品牌认知、转化、招聘、权威、发布支持还是互动

## 核心规则

1. 适应平台。不要跨平台发布相同的内容。
2. 钩子比摘要更重要。
3. 每个帖子应该承载一个明确的想法。
4. 使用具体细节而非口号。
5. 请求要小而明确。

## 平台指南

### X
- 快速切入主题
- 每个帖子或主题帖中的每个推文一个想法
- 除非必要，否则将链接放在正文之外
- 避免标签垃圾

### LinkedIn
- 强有力的第一行
- 短段落
- 更明确地围绕经验、成果和收获进行框架构建

### TikTok / 短视频
- 前 3 秒必须打断注意力
- 围绕视觉脚本，而不仅仅是叙述
- 一个演示、一个声明、一个 CTA

### YouTube
- 尽早展示结果
- 按章节组织结构
- 每 20-30 秒刷新视觉内容

### 新闻通讯
- 提供一个清晰的视角，而不是一堆不相关的项目
- 使章节标题易于浏览
- 让开篇段落发挥真正的作用

## 再利用流程

默认级联：
1. 锚定素材：文章、视频、演示、备忘录或发布文档
2. 提取 3-7 个原子想法
3. 编写平台原生变体
4. 修剪输出中的重复内容
5. 使 CTA 与平台意图保持一致

## 交付物

当被要求做活动时，返回：
- 核心角度
- 平台特定草稿
- 可选的发布顺序
- 可选的 CTA 变体
- 发布前需要的任何缺失输入

## 质量门槛

交付前：
- 每个草稿在其平台上阅读起来很自然
- 钩子有力且具体
- 没有通用的话题语言
- 除非要求，否则不要在平台间复制内容
- CTA 与内容和受众匹配
