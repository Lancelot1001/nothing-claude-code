# 投资者材料

构建一致、可信且易于捍卫的面向投资者的材料。

## 何时激活

- 创建或修改推介文稿
- 撰写投资者备忘录或单页资料
- 构建财务模型、里程碑计划或资金使用表
- 回答加速器或孵化器申请问题
- 围绕单一事实来源对齐多个融资文档

## 黄金法则

所有投资者材料必须相互一致。

在编写前创建或确认单一事实来源：
- 增长指标
- 定价和收入假设
- 融资规模和工具
- 资金用途
- 团队 bios 和头衔
- 里程碑和时间线

如果出现矛盾数字，停止并在起草前解决。

## 核心workflow

1. 盘点规范事实
2. 识别缺失假设
3. 选择资产类型
4. 使用显式逻辑起草资产
5. 将每个数字与事实来源交叉核对

## 资产指导

### 推介文稿
推荐流程：
1. 公司 + 切入点
2. 问题
3. 解决方案
4. 产品 / 演示
5. 市场
6. 商业模式
7. 增长
8. 团队
9. 竞争 / 差异化
10. 请求
11. 资金用途 / 里程碑
12. 附录

如果用户想要原生网页版推介文稿，将此skill与 `frontend-slides` 配对。

### 单页资料 / 备忘录
- 用一句干净的话说明公司做什么
- 展示为什么是现在
- 尽早包含增长和证明点
- 使请求精确
- 保持声明易于验证

### 财务模型
包含：
- 显式假设
- 熊 / 基 / 牛市情况（何时有用）
- 清晰的逐层收入逻辑
- 与里程碑挂钩的支出
- 在决策取决于假设之处的敏感性分析

### 加速器申请
- 回答被问的确切问题
- 优先考虑增长、洞察和团队优势
- 避免夸张
- 内部指标与推介文稿和模型保持一致

## 应避免的红旗

- 无法验证的声明
- 无假设的模糊市场规模
- 不一致的团队角色或头衔
- 不能干净相加的收入计算
- 在假设脆弱处膨胀确定性

## 质量门控

交付前：
- 每个数字与当前事实来源匹配
- 资金用途和收入层正确相加
- 假设可见，而非埋没
- 故事清晰，无炒作语言
- 最终资产在合伙人会议上可辩护
