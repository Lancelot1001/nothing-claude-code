# 设计系统 — 生成和审计视觉系统

## 何时使用

- 开始一个需要设计系统的新项目
- 审计现有代码库的视觉一致性
- 重新设计之前——了解你现有的内容
- 当 UI 看起来"不对劲"但你说不清为什么
- 审查涉及样式的 PR

## 工作原理

### 模式 1：生成设计系统

分析你的代码库并生成一个统一的设计系统：

```
1. 扫描 CSS/Tailwind/styled-components 中的现有模式
2. 提取：颜色、排版、间距、圆角、阴影、断点
3. 研究 3 个竞争对手网站获取灵感（通过浏览器 MCP）
4. 提出一个设计令牌集（JSON + CSS 自定义属性）
5. 生成 DESIGN.md，包含每个决策的理由
6. 创建一个交互式 HTML 预览页面（自包含，无依赖）
```

输出：`DESIGN.md` + `design-tokens.json` + `design-preview.html`

### 模式 2：视觉审计

从 10 个维度对 UI 进行评分（每个 0-10 分）：

```
1. 颜色一致性——你使用的是调色板还是随机十六进制值？
2. 排版层次——清晰的 h1 > h2 > h3 > body > caption？
3. 间距节奏——一致的刻度（4px/8px/16px）还是随意的？
4. 组件一致性——相似的元素看起来相似吗？
5. 响应式行为——流畅还是在断点处出问题？
6. 深色模式——完整还是半成品？
7. 动画——有目的的还是多余的？
8. 无障碍——对比度、焦点状态、触摸目标
9. 信息密度——杂乱还是整洁？
10. 精致度——悬停状态、过渡、加载状态、空状态
```

每个维度都会获得评分、具体示例以及精确到 file:line 的修复建议。

### 模式 3：AI 垃圾检测

识别通用的 AI 生成的设计模式：

```
- 到处使用无意义的渐变
- 紫到蓝的默认色
- 无实际用途的"玻璃拟态"卡片
- 不该圆角的东西使用圆角
- 滚动时过度动画
- 通用 hero 区域配以居中文字和库存渐变
- 毫无个性的无衬线字体栈
```

## 示例

**为 SaaS 应用生成：**
```
/design-system generate --style minimal --palette earth-tones
```

**审计现有 UI：**
```
/design-system audit --url http://localhost:3000 --pages / /pricing /docs
```

**检查 AI 垃圾：**
```
/design-system slop-check
```
