# 命令快速参考

> Claude Code 的 59 个斜杠命令。在任何 Claude Code 会话中输入 `/` 触发。

---

## 核心workflow

| 命令 | 功能 |
|------|------|
| `/plan` | 明确需求、评估风险、编写分步实施计划 — **等待确认后再修改代码** |
| `/tdd` | 强制测试驱动开发：搭建接口 → 写失败测试 → 实现 → 验证 80%+ 覆盖率 |
| `/code-review` | 对修改的文件进行全面的代码质量、安全和可维护性审查 |
| `/build-fix` | 检测并修复构建错误 — 自动委托给正确的构建解决agent |
| `/verify` | 运行完整验证循环：构建 → lint → 测试 → 类型检查 |
| `/quality-gate` | 根据项目标准进行质量门检查 |

| `/rust-test` | Rust 的 TDD（cargo test，集成测试） |

## 快速决策指南

```
开始新功能？         → /plan 先规划，再 /tdd
刚写完代码？          → /code-review
构建失败了？          → /build-fix
需要实时文档？        → /docs <库名>
会话要结束了？        → /save-session 或 /learn-eval
明天继续？            → /resume-session
上下文太重了？         → /context-budget 然后 /checkpoint
想提取学到的东西？     → /learn-eval 然后 /evolve
运行重复任务？         → /loop-start
```
